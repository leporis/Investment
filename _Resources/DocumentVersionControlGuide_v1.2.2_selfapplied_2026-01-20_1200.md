## 1. 버전 번호 체계

```
vX.Y.Z.W

X = 메이저 (Major)   : 문서의 구조나 방향이 크게 바뀔 때
Y = 마이너 (Minor)   : 섹션/주차 단위 내용 추가·수정
Z = 리비전 (Revision): 문단·항목·표 수준의 수정
W = 패치 (Patch)     : 오탈자, 서식, 사소한 수정
```

---

## 2. 파일명 형식

```
주요내용_v버전번호_수정사항키워드_날짜_시간.확장자

※ 버전번호는 점(.)으로 구분: v1.0.0.0, v17.4.2.4 등

예: 빅데이터와사물인터넷_12주강의계획서_v1.0.0.0_초안작성_2026-01-17_0816.docx
예: 교차로시뮬레이터_v17.4.2.4_우회전녹색통과_2026-01-17_1141.html
```

### 이전 형식 (사용하지 않음)
```
❌ 교차로시뮬레이터_v17_4_2_4_우회전녹색통과_2026-01-17_1141.html
```

### 현재 형식 (권장)
```
✅ 교차로시뮬레이터_v17.4.2.4_우회전녹색통과_2026-01-17_1141.html
```

## 3. 사용자가 할 일

### 새 문서 작성 요청 시

그냥 요청하시면 됩니다. Claude가 자동으로 **v1.0.0.0**을 부여합니다.

```
예: "12주 강의계획서를 작성해 주세요"
```

### 문서 수정 요청 시

**수정 내용만 말씀하시면 됩니다.** Claude가 맥락에서 변경 수준을 판단합니다.

```
예: "3주차 내용을 전면 재구성해 주세요"
    → Claude가 마이너 변경으로 판단 → v1.1.0.0

예: "평가 배점을 20%에서 25%로 바꿔주세요"
    → Claude가 리비전 변경으로 판단 → v1.0.1.0

예: "오탈자 수정해 주세요"
    → Claude가 패치로 판단 → v1.0.0.1
```

### 변경 수준을 직접 지정하고 싶을 때

명시적으로 말씀하시면 됩니다.

```
예: "5주차에 활동 추가해 주세요. 마이너 변경으로 처리해 주세요."
```

## 4. Claude가 하는 일

| 단계 | 내용 |
|------|------|
| 1 | 요청 내용에서 변경 수준 판단 |
| 2 | 버전 번호 자동 증가 |
| 3 | 문서 내 버전 번호 업데이트 |
| 4 | 생성/수정 이력 표에 기록 추가 |
| 5 | 새 파일명으로 저장 (버전은 점으로 구분) |
| 6 | 판단이 애매한 경우 안내 또는 확인 요청 |

## 5. 변경 수준 판단 기준

### 메이저 (X 증가)

- 문서 전체 방향 변경
- 프로젝트 주제 변경
- 강좌 구조 전면 개편

```
예: "프로젝트를 스마트 화분에서 교실 환경 모니터로 변경"
    v1.2.3.4 → v2.0.0.0
```

### 마이너 (Y 증가)

- 특정 주차/섹션 전체 재구성
- 새로운 섹션 추가
- 기존 섹션 삭제

```
예: "3주차 내용 전면 재구성"
예: "윤리 토론 섹션 추가"
    v1.2.3.4 → v1.3.0.0
```

### 리비전 (Z 증가)

- 문단, 표, 항목 일부 수정
- 배점, 시간 배분 조정
- 활동 순서 변경
- 용어 통일

```
예: "평가 배점 변경"
예: "6주차 활동 순서 조정"
    v1.2.3.4 → v1.2.4.0
```

### 패치 (W 증가)

- 오탈자 수정
- 날짜, 숫자 오류
- 서식, 띄어쓰기
- 링크 수정

```
예: "오탈자 수정"
예: "날짜 오류 정정"
    v1.2.3.4 → v1.2.3.5
```

## 6. 애매한 경우 처리

Claude가 판단하기 애매한 경우:

### 방법 A: Claude가 판단 후 안내

```
Claude: "이 변경은 리비전 수준으로 판단하여 v1.0.1.0을 적용했습니다.
        다르게 보시면 말씀해 주세요."
```

### 방법 B: Claude가 먼저 확인

```
Claude: "이 변경은 마이너와 리비전 중 어느 수준으로 보시나요?"
```

## 7. 새 대화 시작 시

새 대화에서는 Claude가 이전 버전을 기억하지 못합니다.

**사용자가 할 일:**
- 수정할 문서를 첨부하거나 내용 제공
- 문서에 버전 번호가 포함되어 있으면 Claude가 자동 인식

```
예: "첨부한 강의계획서에서 5주차 내용을 수정해 주세요"
    → Claude가 문서 내 버전(v1.0.0.0)을 읽고 이어서 진행
```

## 8. 세션 인계 시스템

### 8.1 왜 세션 인계가 필요한가?

Claude AI 세션은 다음 상황에서 종료됩니다:
- 컨텍스트 윈도우 한계 도달
- 브라우저 새로고침/닫기
- 장시간 미사용
- 네트워크 오류

세션이 종료되면 **Claude는 이전 대화 내용을 기억하지 못합니다.**

```
┌─────────────────────────────────────────────────────────────────┐
│                    세션 인계의 중요성                            │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  [세션 1]                    [세션 2]                           │
│  • v17.4.2.9까지 작업        • 기억 없음 ❌                     │
│  • 복잡한 알고리즘 논의       • 처음부터 설명 필요               │
│  • 버그 수정 이력             • 같은 버그 다시 발생 가능         │
│                                                                  │
│          인계 문서 없이 → 작업 손실, 시간 낭비                  │
│                                                                  │
│  [세션 1]                    [세션 2]                           │
│  • 인계 문서 생성 ✅          • 인계 문서 제공 ✅                │
│  • 모든 맥락 정리             • 즉시 작업 재개 가능              │
│                                                                  │
│          인계 문서 있으면 → 연속적인 작업 가능                  │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 8.2 세션 인계 문서 구조

세션 종료 전에 다음 구조의 **인계 문서**를 생성합니다:

```markdown
# [프로젝트명] 세션 인계 문서

## 1. 프로젝트 현황
- 현재 버전: vX.Y.Z.W
- 마지막 작업 일시: YYYY-MM-DD HH:MM
- 전체 진행률: XX%

## 2. 파일 목록
| 파일명 | 버전 | 용도 | 상태 |
|--------|------|------|------|
| 파일1.html | v18.0.0.0 | 시뮬레이터 | 완료 |
| 파일2.md | v18.0.0.1 | 설명서 | 완료 |

## 3. 이번 세션 작업 내역
- [완료] 작업1 설명
- [완료] 작업2 설명
- [진행중] 작업3 설명 (XX% 완료)

## 4. 핵심 기술 결정사항
- 결정1: 이유와 함께
- 결정2: 이유와 함께

## 5. 알려진 이슈
- 이슈1: 설명, 우선순위
- 이슈2: 설명, 우선순위

## 6. 다음 세션 작업 목록
- [ ] 다음 작업1
- [ ] 다음 작업2

## 7. 새 세션 시작 가이드
1. 이 문서와 함께 파일 업로드
2. "이전 세션에서 이어서 작업합니다" 입력
3. Claude가 컨텍스트 파악 후 작업 재개
```

### 8.3 세션 인계 요청 방법

작업 중 언제든 다음과 같이 요청하세요:

```
"세션 인계 문서를 만들어주세요"
"다음 세션을 위한 인계 문서를 준비해주세요"
"세션이 끊어질 것 같으니 현재 상태를 정리해주세요"
```

### 8.4 새 세션에서 작업 재개

1. **파일 업로드**: 인계 문서 + 관련 파일들
2. **시작 메시지**:
   ```
   "첨부한 인계 문서를 참고하여 이전 세션에서 이어서 작업합니다.
    다음 작업을 진행해주세요: [작업 내용]"
   ```
3. **Claude 응답**: 컨텍스트 파악 → 작업 재개

### 8.5 자동 인계 트리거

Claude는 다음 상황에서 자동으로 인계 문서 생성을 제안합니다:

| 상황 | Claude 동작 |
|------|------------|
| 대화가 매우 길어짐 | "세션 인계 문서를 생성할까요?" |
| 복잡한 작업 완료 후 | 인계 문서 자동 생성 |
| 사용자 요청 시 | 즉시 생성 |
| 메이저 버전 완료 시 | 인계 문서 자동 생성 |

### 8.6 인계 문서 파일명 형식

```
[프로젝트명]_세션인계_v버전_날짜.md

예: 교차로시뮬레이터_세션인계_v18.0.0.0_2026-01-18.md
```

## 9. 문서 필수 구성요소

Claude가 생성하거나 수정하는 모든 문서에는 다음 구성요소를 **문서 끝에** 포함해야 합니다.

### 9.1 참고자료

문서 작성에 참고한 자료 목록입니다.

```markdown
---

## 참고자료

| # | 자료명 | 유형 | 비고 |
|:-:|--------|------|------|
| 1 | 자료명1 | 문서/웹사이트/파일 등 | 설명 |
| 2 | 자료명2 | 문서/웹사이트/파일 등 | 설명 |

※ 참고한 자료가 없는 경우: "없음" 또는 "해당 없음"으로 표기
```

**포함 대상:**
- 참조한 기존 문서
- 사용자가 제공한 파일
- 웹 검색 결과 (URL 포함)
- 기술 문서, API 문서
- 이전 세션 인계 문서

### 9.2 생성/수정 이력

문서의 버전별 변경 내역입니다.

```markdown
---

## 생성/수정 이력

| 버전 | 날짜 | 시간 | 변경 수준 | 변경 내용 | 작성자 |
|------|------|------|-----------|-----------|--------|
| v1.0.0 | 2026-01-20 | 14:30 | 최초 | 초안 작성 | Claude |
| v1.1.0 | 2026-01-20 | 16:45 | 마이너 | 3장 추가 | Claude |
| v1.1.1 | 2026-01-20 | 17:20 | 리비전 | 표 수정 | Claude |

※ 작성자: Claude, 사용자명, 또는 공동작성 표기
※ 시간: 24시간 형식 (HH:MM)
```

**기록 항목:**
- 버전 번호 (vX.Y.Z 형식)
- 변경 날짜
- 변경 시간 (24시간 형식)
- 변경 수준 (메이저/마이너/리비전/패치)
- 변경 내용 요약
- 작성자

### 9.3 문서 구조 예시

```markdown
# 문서 제목

## 1. 첫 번째 섹션
(내용)

## 2. 두 번째 섹션
(내용)

...

---

## 참고자료

| # | 자료명 | 유형 | 비고 |
|:-:|--------|------|------|
| 1 | 기존설계문서_v1.0.0.md | 문서 | 기본 구조 참고 |

---

## 생성/수정 이력

| 버전 | 날짜 | 시간 | 변경 수준 | 변경 내용 | 작성자 |
|------|------|------|-----------|-----------|--------|
| v1.0.0 | 2026-01-20 | 14:30 | 최초 | 초안 작성 | Claude |
```

### 9.4 적용 범위

| 문서 유형 | 참고자료 | 생성/수정 이력 |
|----------|:--------:|:-------------:|
| 보고서, 기획서 | ✅ 필수 | ✅ 필수 |
| 강의계획서 | ✅ 필수 | ✅ 필수 |
| 기술 문서 | ✅ 필수 | ✅ 필수 |
| 코드 파일 | ⚪ 선택 (주석) | ⚪ 선택 (주석) |
| 세션 인계 문서 | ✅ 필수 | ✅ 필수 |
| 간단한 메모 | ⚪ 선택 | ⚪ 선택 |

## 10. 요약

| 상황 | 사용자 | Claude |
|------|--------|--------|
| 새 문서 작성 | 요청만 | v1.0.0 자동 부여 |

| **문서 완료** | - | **참고자료 + 이력 추가** |
