## 1. 잔고 파일 구조

### 1.1 시트 구성 (4개)

| 시트명 | 용도 | 주요 컬럼 |
|--------|------|-----------|
| **주식잔고** | 보유 종목 현황 | 종목코드, 종목명, 매입가, 보유수량, 현재가, 평가손익, 수익률 |
| **매매이력** | 거래 기록 | 날짜, 종목명, 구분(매수/매도), 수량, 단가, 거래금액, 수수료, 실현손익, 수익률, 비고 |
| **손익요약** | 투자 손익 요약 | 총 실현손익, 평가손익, 투자원금, 현재 평가액, 총 수익률, 월별 실현손익 집계(F~I열) |
| **실현손익차트** | 차트용 데이터 | 일자, 종목명, 실현손익 |

### 1.2 매매이력 시트 레이아웃

```
1~3행: 안내 문구
6행:   기존 거래 (월초 누적)
7행:   (빈 행)
8행:   헤더
9행~:  매매이력 (날짜순)
마지막: 월별 소계 → 실현손익 합계
```

## 3. 매매이력 업데이트

매매 발생 시 잔고 파일의 **4개 시트를 동시에** 업데이트합니다.

### 3.1 주식잔고 시트

- 보유현황 업데이트 (수량, 평균매입가 등)
- **0주 종목은 행 삭제**
- 신규 종목은 행 추가

### 3.2 매매이력 시트

- 월별 실현손익 소계 행 **위에** 매매횟수만큼 행 삽입
- 매매이력 입력 (날짜순 정렬)
- 실현손익 색상: 수익(+) 빨간색, 손실(-) 파란색

### 3.3 손익요약 시트

- 종목별 실현손익 집계 업데이트
- 월별 실현손익 집계 (F~I열: 월, 매매건수, 실현손익, 누적손익) 업데이트

### 3.4 실현손익차트 시트

- 총실현손익 행 **위에** 새 행 삽입
- 형식: A열(일자) | B열(종목명) | C열(실현손익)
- 총실현손익 행: A열("총실현손익") | B열(빈칸) | C열(합계, 굵은글씨)

### 3.5 보유종목 변동 시 추가 처리

- 메모리 자동 업데이트 (보유종목 목록)
- 잔고파일 업로드 시 메모리와 비교하여 차이 알림

## 4. 종가 업데이트

당일 종가 입력 시 다음 항목을 업데이트합니다.

### 4.1 주식잔고 시트

- 각 종목의 현재가 갱신
- 평가손익, 수익률 자동 재계산

### 4.2 손익요약 시트

| 항목 | 설명 |
|------|------|
| 평가손익 | 현재 보유 중인 종목의 미실현 손익 |
| 투자원금 | 총 매입금액 |
| 현재 평가액 | 현재가 × 보유수량 합계 |
| 총 수익률 | (현재 평가액 - 투자원금) / 투자원금 × 100 |

---

## 5. 월별 마감 처리

### 5.1 월말 마감

- 해당 월 소계 행 확정
- 손익요약 시트 월별 데이터 확정

### 5.2 새 달 시작

- 새 월 매매이력 영역 생성
- 월별 소계 행 신규 추가

## 6. 서식 규칙

|------|------|

## 7. 작업 요청 예시

### 7.1 매매이력 입력

**기본 형식:**
```
[잔고 파일 업로드]

오늘(2/6) 매매이력 입력해줘:
- 삼성전자 10주 매수 85,000원
- 하나마이크론 100주 매도 26,500원 (일부 익절)
```

**상세 형식 (매입가 지정 필요 시):**
```
[잔고 파일 업로드]

오늘(2/6) 매매이력 입력해줘:

매수:
- 삼성전자 10주 85,000원 (신규 매수)

매도:
- 하나마이크론 100주 26,500원, 매입가 24,845원 (일부 익절)
```

**당일 매수/매도 (단타):**
```
오늘(2/6) 매매이력:
- 루미르 400주 매수 13,380원
- 루미르 400주 매도 15,360원 (단기 익절)
```

### 7.2 종가 업데이트

**직접 입력 (권장):**
```
[잔고 파일 업로드]

종가 업데이트:
- 하나마이크론: 26500
- 코오롱인더: 48000
- 필옵틱스: 46050
...
```

**자동 조회 요청:**
```
[잔고 파일 업로드]

보유 종목 종가 한 번에 조회해서 업데이트해줘
```

### 7.3 월별 마감

```
[잔고 파일 업로드]

1월 매매이력 마감 처리해줘:
- 1월 소계 행 추가
- 손익요약 시트 1월 데이터 확정
```

### 7.4 종합 요청 (일일 루틴)

```
[잔고 파일 업로드]

오늘(2/6) 잔고 정리:

1. 매매이력
   - 하나마이크론 100주 매도 26,500원 (일부 익절)

2. 종가
   - 하나마이크론: 26500
   - 코오롱인더: 48000
   - (나머지 종목...)

3. 업데이트된 잔고 파일과 일일 리포트 만들어줘
```

### 7.5 기타

```
# 신규 종목 추가
신규 종목 추가: 종목코드 005930, 삼성전자 10주, 매입가 85,000원

# 전량 매도 후 삭제
전량 매도 완료된 한국무브넥스 주식잔고에서 삭제해줘

# 손익 현황 / 수익률 순위 / 특정 종목 매매이력
현재 손익 현황 요약해줘
수익률 상위 5개, 하위 5개 종목 알려줘
다날 매매이력 전체 보여줘
```

## 8. 공통 규칙

### 8.1 현재가 기준

| 시점 | 현재가 기준 |
|------|------------|
| 장 종료 후 (15:30~) | 당일 종가 |
| 장 종료 전 (~15:30) | 전일 종가 |

⚠️ **모든 종목 현재가는 웹 검색으로 재확인 필수**

### 8.2 숫자 정확성

> ⚠️ **숫자 값의 정확성이 매우 중요**
>
> - 모든 수치는 최신 데이터를 웹 검색으로 재확인
> - 신중하게 계산
> - **숫자 오류는 심각한 피해 초래**

### 8.3 파일 수정 후 검증

> 📋 **파일 수정 후 반드시 실제 파일을 열어 수정 내용을 검증한 후 결과 보고**
>
> - 작업 완료 후 즉시 검증
> - 예상과 실제 결과 비교
> - 정확한 상태만 보고

### 8.4 문서 업로드 시 확인

- 포트폴리오 관련 문서 업로드 시 최종 수정일이 현재 날짜보다 **1개월 이상** 지났으면 자동으로 알림

### 8.5 잔고 파일 수정 범위

- 잔고 엑셀 파일은 **"계좌 현황" 대화에서만** 수정
- 다른 대화에서는 잔고 파일을 수정하지 않고 안내만 제공
